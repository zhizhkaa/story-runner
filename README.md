# Story Runner

Небольшой сервис для совместного прогона User Story на Android и iOS. Участники работают без регистрации: вводят имя, берут свободные разделы или отдельные пункты, сохраняют результаты и при необходимости пропускают пункт для следующего исполнителя.

## Быстрый запуск

Требуется Docker Desktop.

```bash
docker compose up --build
```

После запуска:

- главная страница: [http://localhost:8001](http://localhost:8001);
- панель управления: [http://localhost:8001/manage/](http://localhost:8001/manage/);
- пароль панели по умолчанию: `change-me`.

Данные сохраняются в `data/story-runner.sqlite3` и не пропадают после перезапуска контейнера.

## Настройка

Для локальной настройки скопируйте пример окружения:

```bash
cp .env.example .env
```

Основные переменные:

| Переменная | Назначение |
|---|---|
| `ADMIN_PASSWORD` | Единый пароль панели управления |
| `SECRET_KEY` | Секрет Django для сессий и CSRF |
| `DEBUG` | `1` для локальной разработки, `0` для сервера |
| `ALLOWED_HOSTS` | Разрешённые имена хостов через запятую |
| `CSRF_TRUSTED_ORIGINS` | Доверенные HTTPS-адреса через запятую |

Перед размещением в сети обязательно замените `ADMIN_PASSWORD` и `SECRET_KEY`. В режиме `DEBUG=0` сервис откажется запускаться с тестовыми значениями.

## Локальная разработка без Docker

```bash
uv venv .venv
uv pip install --python .venv/bin/python -r requirements.txt
.venv/bin/python manage.py migrate
.venv/bin/python manage.py runserver 8001
```

Проверка:

```bash
.venv/bin/python manage.py test
.venv/bin/python manage.py check
```

Стили собираются Tailwind CSS CLI:

```bash
npm ci
npm run css:build
```

## Основной сценарий

1. Администратор входит в `/manage/`, при необходимости меняет текстовый шаблон с помощью отступов и создаёт активный прогон с ОС, версией и сборкой.
2. Участник открывает главную страницу, выбирает активный прогон, вводит имя и берёт разделы, подразделы или отдельные пункты.
3. Для каждого пункта участник выбирает «ОК», «НЕ ОК» или «Пропустить». Пропущенный пункт освобождается, а причина и автор сохраняются в истории.
4. Когда все конечные пункты получили результат, прогон автоматически попадает в архив. Администратор также может завершить его досрочно со статусом «НЕ ОК».
5. Состояние активного прогона и его итог доступны по одной публичной ссылке с превью для мессенджеров.
6. Архив группируется по ОС, версии и сборке. История раскрывается на главной странице и доступна по отдельной постоянной ссылке.
