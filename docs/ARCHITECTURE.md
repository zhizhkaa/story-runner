# Архитектура Story Runner

## Состав

- Django отдаёт серверные HTML-страницы, проверяет права и выполняет транзакции.
- SQLite хранит шаблон, снимки прогонов, результаты, назначения и историю пропусков.
- Gunicorn запускает приложение в контейнере.
- WhiteNoise раздаёт статические файлы.
- Tailwind CSS CLI собирает стили из классов в Django-шаблонах без JavaScript во время работы сервиса.

Внешнего JSON API и пользовательских аккаунтов в первой версии нет.

## Модель данных

- `StoryTemplate` — редактируемый шаблон.
- `TemplateNode` — узел шаблона: раздел или проверяемый пункт.
- `StoryRun` — прогон для конкретных ОС, версии и сборки.
- `RunNode` — неизменяемый снимок узла в прогоне и его конечный результат.
- `Participant` — имя и хеш секретного браузерного ключа.
- `Claim` и `ClaimItem` — текущее назначение свободных конечных пунктов.
- `DraftResult` — черновик участника.
- `SkipEvent` — неизменяемая запись о пропуске.

При активации дерево шаблона копируется в `RunNode`. Поэтому изменение шаблона не затрагивает активные и завершённые прогоны.

Редактор принимает обычный текст: каждая непустая строка становится узлом, а ведущие пробелы определяют родителя. Разделом считается строка, у которой есть вложенные строки. Нумерация рассчитывается при сохранении: первые два уровня числовые, третий — с кириллическими буквами, последующие снова числовые.

## Конкурентная работа

Каждый активный `ClaimItem` имеет уникальную связь с конечным пунктом. Создание назначения выполняется в транзакции: если другой запрос успел забрать пункт, база отклоняет повторную запись, а пользователь получает понятное сообщение.

При отправке назначения итоговые результаты записываются в снимок, а пропущенные пункты освобождаются. Прогон завершается только тогда, когда у каждого конечного пункта есть `ОК` или `НЕ ОК`.

## Защита

- публичные участники не проходят авторизацию;
- их браузерный ключ хранится в `HttpOnly`, `SameSite=Lax` cookie, а в базе хранится только SHA-256-хеш;
- чужое назначение нельзя открыть по известной ссылке без соответствующего cookie;
- панель управления защищена отдельной сессией и паролем из окружения;
- все изменяющие формы используют POST и CSRF-защиту;
- завершённые снимки через интерфейс не редактируются.
